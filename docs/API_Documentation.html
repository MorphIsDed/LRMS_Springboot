1. Overview & Architecture

The LRMS Partner API enables third-party service providers such as Zomato, Swiggy, and Booking.com to programmatically interact with the LRMS platform. The API follows RESTful conventions and is versioned under /api/v1/partners/.

Base URLhttp://localhost:8081/api/v1/partners
Auth MethodAPI Key via X-API-KEY header (SHA-256 hashed, stored in DB)
Content Typeapplication/json
Tech StackJava 21, Spring Boot 3.2, PostgreSQL, Flyway

Architecture Diagram

2. Authentication

All partner endpoints require a valid X-API-KEY header. The raw key is SHA-256 hashed server-side and matched against the api_keys table.

Test Key (Development): X-API-KEY: test_zomato_key_12345

Request Header

GET /api/v1/partners/restaurant/menu HTTP/1.1
Host: localhost:8081
X-API-KEY: test_zomato_key_12345

Unauthorized Response (401)

{
  "success": false,
  "message": "Missing X-API-KEY header"
}

3. Standard Response Envelope

All partner API responses are wrapped in a standard envelope:

{
  "success": true,
  "data": { ... },
  "message": "Operation completed successfully"
}

4. Lodging Endpoints

GET /api/v1/partners/lodging/rooms

Description: List all rooms, optionally filtered by availability.

Query Parameters (all optional)

NameTypeDescription
checkInLocalDateCheck-in date (ISO format: 2026-06-01)
checkOutLocalDateCheck-out date
typeEnumSINGLE, DOUBLE, SUITE, DELUXE
guestsShortNumber of guests

Sample Response (200 OK)

{
  "success": true,
  "data": [
    {
      "id": 1,
      "roomNumber": "101",
      "roomType": "DELUXE",
      "floor": 1,
      "baseRate": 4500.00,
      "status": "AVAILABLE",
      "maxOccupancy": 3
    }
  ],
  "message": "Rooms fetched successfully"
}
POST /api/v1/partners/lodging/bookings

Description: Create a new booking. Guest identity is auto-resolved from API key.

Request Body

{
  "roomId": 1,
  "checkIn": "2026-06-15",
  "checkOut": "2026-06-18",
  "guestName": "Rahul Sharma",
  "guestEmail": "rahul@example.com"
}

Response (200 OK)

{
  "success": true,
  "data": {
    "id": 5,
    "status": "RESERVED",
    "checkIn": "2026-06-15",
    "checkOut": "2026-06-18",
    "totalRoomCharges": 13500.00
  },
  "message": "Booking created successfully"
}
PATCH /api/v1/partners/lodging/bookings/{id}

Description: Update booking dates. Only RESERVED bookings can be modified. Re-calculates pricing automatically.

Query Parameters

NameTypeDescription
checkInLocalDateNew check-in date
checkOutLocalDateNew check-out date
PATCH /api/v1/partners/lodging/bookings/{id}/cancel

Description: Cancel a booking. Sets status to CANCELLED (soft delete). Room is freed.

5. Restaurant Endpoints

GET /api/v1/partners/restaurant/menu

Description: List all available menu items, optionally filtered by category.

Query Parameters (optional)

NameTypeDescription
categoryEnumSTARTER, MAIN, DESSERT, BEVERAGE

Sample Response (200 OK)

{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "Paneer Tikka",
      "category": "STARTER",
      "price": 350.00,
      "isAvailable": true,
      "inventoryCount": 25
    }
  ],
  "message": "Menu items fetched successfully"
}
POST /api/v1/partners/restaurant/orders

Description: Place a new restaurant order. Inventory is auto-deducted. Tax (5%) calculated server-side.

Request Body

{
  "tableNumber": "Table 5",
  "items": [
    { "itemId": 1, "quantity": 2 },
    { "itemId": 3, "quantity": 1 }
  ]
}

Response (200 OK)

{
  "success": true,
  "data": {
    "id": 12,
    "status": "PENDING",
    "subtotal": 1050.00,
    "taxAmount": 52.50,
    "totalAmount": 1102.50
  },
  "message": "Order placed successfully"
}
PATCH /api/v1/partners/restaurant/orders/{id}

Description: Update order status.

Request Body

{ "status": "PREPARING" }

Valid statuses: PENDING, PREPARING, READY, SERVED, PAID, CANCELLED

PATCH /api/v1/partners/restaurant/orders/{id}/cancel

Description: Cancel an order. Sets status to CANCELLED.

6. Admin Dashboard Endpoints

Note: These endpoints are secured by JWT (Admin role only), NOT by API Key.
GET/api/admin/partner-stats/summary?days=30

Returns aggregated usage metrics.

{ "totalCalls": 1247, "errorCount": 23, "errorRate": 1.84, "activePartners": 3 }
GET/api/admin/partner-stats/usage-by-partner?days=30

Returns per-partner request and error counts.

GET/api/admin/partner-stats/recent-logs

Returns the 50 most recent API request logs with method, endpoint, status, and latency.

8. Database Schema (Flyway V3)

CREATE TABLE api_keys (
    id            BIGSERIAL PRIMARY KEY,
    key_hash      VARCHAR(64) NOT NULL UNIQUE,
    partner_name  VARCHAR(100) NOT NULL,
    is_active     BOOLEAN NOT NULL DEFAULT TRUE,
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    system_user_id BIGINT NOT NULL REFERENCES users(user_id)
);

CREATE TABLE api_usage_logs (
    id              BIGSERIAL PRIMARY KEY,
    api_key_id      BIGINT NOT NULL REFERENCES api_keys(id),
    method          VARCHAR(10) NOT NULL,
    endpoint        VARCHAR(255) NOT NULL,
    http_status     INTEGER NOT NULL,
    response_time_ms BIGINT NOT NULL,
    requested_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
